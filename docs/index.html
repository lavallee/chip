Chips are packaged units that observe declared signals, apply bounded — optionally LLM-backed — judgment, and produce policy-gated responses with receipts. Chips compose into linear circuits; hosts bind and run them.

What is a chip

This repository is the contract itself — the spec, a validation library, and a conformance kit. It is not a runtime, and it has zero runtime dependencies.

Port

A declared input or output. A chip states up front which ports it reads and which it writes — no implicit signal reads, no surprise side channels.

Stage

A chip running its bounded contract: observe the ports it declared, apply judgment within its own scope, emit a receipt. One promise per stage.

Circuit

A linear sequence of stages. Each stage consumes the prior stage's ports and produces its own — composition without hidden fan-out.

Binding

The wiring a host supplies at run time: which real data fills which port, and under what policy, for one circuit.

Receipt

The record of a run — including the runs where a chip observed and decided not to act. No silent chips.

Host and gateway

A host binds and runs circuits, enforces policy, and persists receipts — fab is a reference implementation. A chip that needs LLM-backed judgment reaches it through a gateway rather than holding its own credentials — somm is a reference implementation. The owning system is whatever product or team the host ultimately serves.

Install

# contract + validation only:
$ pip install chipspec

# + schema validation for port payloads:
$ pip install "chipspec[jsonschema]"

Python 3.12+. The import package is chip — the distribution is named chipspec on PyPI because chip is squatted.

Design principles

  • One promise per chip. A chip declares one bounded job and does only that job.
  • Deterministic envelope around probabilistic judgment. LLM-backed decisions live inside a fixed, inspectable shape — ports, policy gates, and receipts don't get to be probabilistic too.
  • Authority fails closed. No implicit escalation. Absent explicit policy grant, a chip acts as if it has none.
  • Receipts for every run, including quiet ones. A chip that observes and decides not to act still produces a receipt saying so.
  • Taint is transitive. Anything a chip's judgment touches carries forward whatever provenance or trust constraints it entered with.
  • Targets own final deduplication. A chip proposes; whatever system it writes into is the one source of truth for whether that write is new.
